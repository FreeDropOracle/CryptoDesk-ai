# Beta Success Metrics Dashboard

Track these metrics weekly during the closed beta.

## Core Targets

| Metric | Target | Current | Status |
|--------|--------|---------|--------|
| Active Beta Users | 20 to 50 | 0 | Not Started |
| Weekly Retention | 60%+ | 0% | Not Started |
| Onboarding Completion | 80%+ | 0% | Not Started |
| Critical Bugs Open | 0 | 0 | On Track |
| User Satisfaction | 4/5+ | 0/5 | Not Started |

## Weekly Checkpoints

### Week 1

- [ ] First cohort onboarded
- [ ] First feedback collected
- [ ] First triage review completed

### Week 2

- [ ] Major confusion points identified
- [ ] First fix release shipped if needed
- [ ] Survey results summarized

### Week 3

- [ ] Retention trend reviewed
- [ ] AI trust feedback reviewed
- [ ] Translation quality issues categorized

### Week 4

- [ ] Mid-beta go / no-go review completed
- [ ] Public-launch blockers listed
- [ ] Cohort expansion decision made

## Suggested Data Sources

- Applicant tracker
- Discord support and bug-report counts
- Weekly survey results
- Internal release notes and fix logs
