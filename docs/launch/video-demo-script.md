# Video Demo Script

Use this as the master script for the first CryptoDesk AI beta demo video.

## Goal

Explain the product clearly, reduce trust friction, and guide interested testers toward the beta signup flow.

## Audience

- beta applicants
- simulation-first traders
- security-conscious desktop users
- early adopters evaluating the product before download

## Format

- Length: 5 to 7 minutes
- Style: screen recording plus voiceover
- Output: one full demo, plus shorter cutdowns for social channels

## Segment 1: Hook

Duration: 0:00 to 0:45

Visuals:

- logo or wordmark
- short opening animation or polished static title card
- fast cuts of dashboard, AI card, and simulation screen

Voiceover direction:

- open with the promise of clarity and safety
- frame the product as a non-custodial desktop co-pilot

Suggested English line:

> Crypto trading should be clearer, safer, and more transparent. This is CryptoDesk AI, a desktop workspace built around simulation-first workflows and explainable AI.

Suggested Arabic line:

> تداول العملات الرقمية يحتاج إلى وضوح وأمان أكثر. هذا هو CryptoDesk AI، مساحة عمل مكتبية مبنية على المحاكاة أولاً وتوصيات ذكاء اصطناعي قابلة للتفسير.

On-screen text:

- Non-Custodial
- Simulation First
- Explainable AI
- Desktop First

## Segment 2: The Problem

Duration: 0:45 to 1:30

Visuals:

- contrast cluttered exchange screens with the cleaner CryptoDesk AI flow
- highlight confusing live trading controls from typical platforms

Voiceover direction:

- explain that many tools are noisy, difficult to trust, and weak on user education
- mention that security is often treated as an afterthought

On-screen prompts:

- Complex UI
- Black-box tools
- Weak user trust

## Segment 3: The Product Walkthrough

Duration: 1:30 to 3:30

Visuals:

- dashboard with market updates
- AI signal card with reasoning
- simulation page and virtual portfolio
- settings page showing bilingual support and safe preferences

Voiceover direction:

- show the product as one connected workflow
- emphasize that the AI explains itself
- show that the user can practice safely before real execution is even considered

Feature talking points:

- live market visibility
- explainable AI signals
- simulation trading with clear safeguards
- English and Arabic support
- onboarding with required risk acknowledgment

## Segment 4: Security Deep Dive

Duration: 3:30 to 4:45

Visuals:

- simple layered diagram
- settings and security modal
- key architectural labels only, no real credentials

Voiceover direction:

- explain that keys remain local
- note the preload boundary and validated IPC
- make clear that the renderer does not get direct secret access

On-screen diagram:

```text
User Device
├─ OS Keychain
├─ Local Encryption
├─ Validated IPC Boundary
└─ Direct Exchange Connectivity
```

Key points:

- no remote custody
- no direct renderer access to secrets
- live trading remains intentionally gated in the current release track

## Segment 5: Beta Invitation

Duration: 4:45 to 5:30

Visuals:

- beta signup page
- docs or support materials
- Discord or community placeholder

Voiceover direction:

- invite careful testers, not hype-driven signups
- explain that the beta is small and feedback-oriented

On-screen text:

- Closed Beta
- Limited Cohort
- Apply: `cryptodesk.ai/beta`
- Watch Demo: `cryptodesk.ai/demo`

## Segment 6: Legal And Risk Disclaimer

Duration: 5:30 to end

Visuals:

- calm end screen
- concise legal copy

Voiceover direction:

- end clearly and responsibly
- remind viewers that this is not financial advice
- encourage simulation-first use

On-screen disclaimer:

- Not Financial Advice
- Non-Custodial
- Use Simulation First
- Beta Software May Contain Bugs

## Visual Safety Rules

- never show real API keys
- never show personal desktop files or notifications
- never imply automatic profits or guaranteed outcomes
- never describe the current beta as live auto-trading software

## Recording Notes

- capture both English and Arabic UI states if possible
- if using one language in the main video, provide subtitles for the other
- keep cursor movement calm and intentional
- avoid empty states unless you are explicitly explaining them
