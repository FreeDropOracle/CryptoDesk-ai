# Launch Day Runbook

Use this checklist for the day the release or public beta goes live.

## T-24 Hours

- [ ] Final build validated
- [ ] Release notes reviewed
- [ ] Download links checked
- [ ] Announcement drafts approved
- [ ] Support contacts aware of the launch window
- [ ] Community moderators briefed

## T-1 Hour

- [ ] Release page ready but not yet public
- [ ] Demo link confirmed working
- [ ] Community invite verified
- [ ] Email platform prepared
- [ ] Social posts loaded and ready

## Launch Moment

- [ ] Publish release or beta access page
- [ ] Publish master announcement
- [ ] Publish X / Twitter thread
- [ ] Publish Discord announcement
- [ ] Send launch email
- [ ] Start monitoring incoming questions and bugs

## T+2 Hours

- [ ] Review first user questions
- [ ] Check for broken links or install blockers
- [ ] Note any language or messaging confusion
- [ ] Escalate critical issues immediately

## T+24 Hours

- [ ] Summarize downloads, signups, and first feedback
- [ ] Publish a short thank-you update
- [ ] Prioritize any urgent fixes
- [ ] Decide whether promotion should continue, pause, or narrow

## Launch Commandments

- communicate clearly
- under-promise and over-explain
- do not hide issues
- fix trust problems before growth problems
