# Discord Server Setup For Beta

Use a small, structured server. The goal is signal quality and support clarity, not noise.

## Category: Announcements

- `#welcome`
  - rules
  - support expectations
  - simulation-first reminder
- `#announcements`
  - official updates only
- `#changelog`
  - version notes and beta fixes

## Category: Community

- `#general-chat`
- `#trading-discussion`
- `#showcase`

## Category: Beta Feedback

- `#bug-reports`
- `#feature-requests`
- `#beta-feedback`

## Category: Support

- `#faq`
- `#help-desk`
- `#technical-issues`

## Category: Staff Only

- `#staff-chat`
- `#beta-applicants`
- `#bug-triage`

## Setup Notes

- Keep reporting templates pinned in `#bug-reports`
- Keep the current build number pinned in `#announcements`
- Lock `#announcements` and `#changelog` to staff only
- Make `#welcome` the first channel every tester sees
- Add one pinned message clarifying that the beta is simulation-first and advisory-only
